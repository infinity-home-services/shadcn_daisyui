Platforms

Dual units, HIG arbitration, and the token map between web and SwiftUI.

Portable source: usage-rules/foundations-platforms.md · all-in-one bundle

One value, every platform

Values are stated in dual units everywhere: 1rem = 16px on the web and 1pt ≈ 1px at 1× on iOS - so a value written once is the same physical size on both.

0.5rem

8px · 8pt

1rem

16px · 16pt

1.5rem

24px · 24pt

2.75rem

44px · 44pt

Who wins: HIG or these guidelines?

On iOS, Apple's Human Interface Guidelines own the platform chrome; this design system owns the brandable surface. Never invent a third option.

Area Winner Examples
Navigation chrome HIG Tab bars, nav bars, split views, back button
System gestures & surfaces HIG Swipe-back, alerts, share sheets, keyboards
Spacing scale Guidelines 4pt grid, blessed steps
Type roles & weights Guidelines The six-role ramp, mapped to SF text styles
Color semantics & radius Guidelines Semantic roles, 6/8/10/14pt corners
Touch target floor Both agree 44 × 44pt minimum

Token map → SwiftUI

The ShadcnDaisyUI Swift package ships these as Color/constant tokens with light + dark variants baked in.

Web token Swift token System analogue

--foreground Color.sdForeground label
--muted-foreground Color.sdMutedForeground secondaryLabel
--primary Color.sdPrimary accent color role
--destructive Color.sdDestructive systemRed role
--border-color Color.sdBorder separator
--radius-xl (14px) Radius.xl (14pt) continuous corners
